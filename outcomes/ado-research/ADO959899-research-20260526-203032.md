### Code Questions
#### 1. Where does LinkCentral expose the machine feedback/update and feedback/complete APIs used by the Goepfert link?

The REST endpoints are in `src/main/java/com/kiwiplan/link/linkcentral/controller/LineupController.java` under the `/api/v1` request mapping:

- `updateJob(...)` handles `POST /api/v1/machines/{machineNumber}/feedback/update` and delegates to `LineupEntryService.updateJobProgress(machineNumber, updateReqDTO)`.
- `completeJob(...)` handles `POST /api/v1/machines/{machineNumber}/feedback/complete` and delegates to `LineupEntryService.finishJob(machineNumber, updateReqDTO)`.

The OpenAPI source of truth in `api/linkcentral.yaml` also documents these as `/api/v1/machines/{machineNumber}/feedback/update` and `/api/v1/machines/{machineNumber}/feedback/complete`. This differs from some prior ADO/domain notes that mention `/api/v2/.../feedback/...`; for this repository snapshot, the feedback endpoints are `/api/v1`.

For this support item, repeated HTTP 500s would most likely originate after `LineupController` delegates into the active `LineupEntryService` implementation, not from routing itself.

#### 2. What service logic processes feedback/update and feedback/complete for classic/PCS behavior, and where can failures turn into HTTP 500 responses?

`LineupEntryService` defines the two relevant operations: `updateJobProgress(...)` and `finishJob(...)`.

For classic behavior, `src/main/java/com/kiwiplan/link/linkcentral/service/impl/classic/LineupEntryClassicService.java` implements both:

- `updateJobProgress(...)` validates setup/start time, fetches a `WORKIP` record, creates or updates a `FACTRY` record, updates `WORKIP`, and wraps the work through `ClassicDbService.insertRowToDb(...)` with locks on `WORKIP`, `FACTRY`, `JBSPEC`, and `STEDIE`.
- `finishJob(...)` updates `FACTRY` complete data and deletes the matching `WORKIP` record through `ClassicDbService.accessDb(...)` with locks on `WORKIP` and `FACTRY`.
- If those locked operations return `null`, both methods throw `InternalServerErrorException("Failed to update the job progress...")`.
- `ClassicDbService` also converts lock `IOException` or `TimeoutException` into `InternalServerErrorException("Failed to acquire database lock: ...")`.

For CSC-only refresh behavior, `src/main/java/com/kiwiplan/link/linkcentral/service/impl/csconlyrefresh/LineupEntryCscOnlyRefreshService.java` routes by machine type:

- converter updates delegate back to `LineupEntryClassicService.updateJobProgress(...)` via `updateConverterJobProgress(...)`.
- converter completion uses `finishConvertorJob(...)`, which also updates `FACTRY`, deletes `WORKIP`, and can throw `InternalServerErrorException` if the locked DB operation returns `null`.
- corrugator feedback is sent through `sendCorrugatorFeedbackRequest(...)`; a null or unsuccessful comms response throws `InternalServerErrorException("Failed to send request")`.

`AppExceptionHandler.handleAllExceptions(...)` catches otherwise unhandled exceptions and returns `INTERNAL_SERVER_ERROR` with HTTP 500. That means DB lock failures, missing/unexpected runtime exceptions, and explicit `InternalServerErrorException` paths can all present to LinkDevice/XMGEN as 500 responses.

#### 3. Which configuration decides the active LinkCentral service mode and therefore which feedback implementation is used for this customer?

`src/main/java/com/kiwiplan/link/linkcentral/config/AppServicesConfig.java` creates the active `LineupEntryService` based on `EnvContext.getEnv()`:

- `CLASSIC` selects `LineupEntryClassicService`.
- `VUE` selects `PcsCscApiService`.
- `CSC_ONLY_REFRESH` selects `LineupEntryCscOnlyRefreshService`.

`src/main/java/com/kiwiplan/link/linkcentral/config/env/EnvContext.java` decides the environment as follows:

- If `kiwiplan.env=classic`, the environment is forced to `CLASSIC`.
- If `kiwiplan.env` is absent and `kiwiplan.mes.service-url` is configured, the environment is inferred from license products/features.
- If products include `csc` and `pcs` without `classic.pcs`, env becomes `VUE`.
- If products include `csc` and either do not include `pcs` or include feature `classic.pcs`, env becomes `CSC_ONLY_REFRESH`.

The Stora Goepfert sample config at `src/main/resources/sample/linkcentral.yaml-gopfert-stora` sets database URLs, LinkCentral port `8444`, comms configuration, and `kiwiplan.mes.service-url`, while `kiwiplan.env: classic` is commented out. That means a deployed Stora-like setup may rely on license inference and can land in `CSC_ONLY_REFRESH`, where converter feedback still delegates to classic DB update logic but corrugator branches use comms feedback. Confirm the runtime log line `Building LineupEntryService for env ...` before assuming which implementation handled the failing request.
